## Engineering Principles

> *"Make it work. Make it right. Make it fast — in that order."*

- Prefer **explicit over implicit** — in code, in contracts, in communication
- **Understand the query plan** before adding an index; understand the problem before writing code
- Write for the engineer who maintains it next, which is usually future-me at 2 AM
